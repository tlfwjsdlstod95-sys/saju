## AI 1:1 상담 챗 설계 메모

## 토스페이먼츠 결제 실연동 설계 메모 (테스트 모드)
- **흐름**: Paywall `pay()` → 토스 SDK v2(`js.tosspayments.com/v2/standard`) `requestPayment`(redirect) → 성공 시 `/payment/success?paymentKey&orderId&amount` → 클라가 `POST /api/payment/confirm` → 서버가 토스 `v1/payments/confirm`(Basic auth, secret키) 승인 → 성공하면 `localStorage saju_premium_v1=1` 세팅 후 원래 경로로 복귀.
- **금액 위변조 방지**: confirm 라우트가 `PRODUCT_AMOUNT=9900` 하드코딩과 대조. 클라가 보낸 금액 신뢰 안 함.
- **복귀 경로**: 결제 전 `localStorage saju_pay_return=pathname` 저장 → success 페이지가 읽어 복귀(메인/궁합 공용).
- **키**: `.env.local` `NEXT_PUBLIC_TOSS_CLIENT_KEY`(test_ck_...) / `TOSS_SECRET_KEY`(test_sk_...). 현재 토스 공개 문서 테스트 키. **운영 전환 = 사업자등록+가맹점 계약 후 본인 키로 교체만** 하면 됨(코드 변경 불필요).
- **주의**: 기존 '데모 즉시 해제'는 제거됨. Paywall의 `onUnlock` prop은 호환 위해 남았으나 미사용(redirect가 해제 담당). 테스트: 토스 결제창에서 아무 카드번호 입력해도 실청구 없음. 승인은 결제 후 10분 내 필요.
- 샌드박스에선 토스 API 직접 호출이 막혀 실승인까지는 사용자 브라우저에서 확인 필요(검증은 금액/필수값 400 경로까지 완료).
- **영수증/이용내역**: confirm 라우트가 `orderName/isTest`까지 반환 → success 페이지가 `addReceipt()`로 `saju_receipts_v1`에 저장. `Receipts.tsx`(영수증 없으면 null 반환)가 프리미엄 카드 아래 표시, 행 클릭 시 영수증 상세 펼침. 서버 DB 아님(localStorage). 실서비스 전환 시 confirm 라우트 주석 위치에 DB 기록 추가.

## 오늘의 운세 설계 메모
- `computeDailyFortune(result, date=오늘)`: 오늘 일진(JDN→60갑자)을 사용자 일간·신강약에 대입. `favor()`(신강=설기 길, 신약=인비 길, 중화 완만) → 총운 raw, `mapScore()`로 0~100 완만 매핑. 카테고리(애정/재물/일·공부/건강)는 오늘 두 오행의 십신그룹이 타깃이면 favor만큼 가감. 같은 날·같은 사주는 `seed`로 문구/행운 고정.
- 행운 색/숫자/방위 = 신강약 기준 용신프록시 오행(신약 인성·신강 식상·중화 재성)에서 도출.
- 클라 전용(`DailyFortune.tsx`, useMemo, API 0원). 자정에 일진 바뀜(todayKey).
- `scripts/_daily_test.ts`는 임시 검증 잔재(내용 비움). 샌드박스가 마운트 unlink 불가라 삭제 못함 — 사용자가 지워도 무방.

## 신년운세 설계 메모
- `computeYearlyFortune(result, year)`: **핵심 트릭 — 각 월 15일은 항상 절입 이후라 월지가 그레고리력 월로 고정**(`MONTH_BRANCH=[1,2..11,0]`). 그래서 VSOP/절기 없이 월두법 상수(`MONTH_STEM_START`)만으로 12개월 월간지 도출 → 클라 번들 가벼움(검증: computeSaju 15일과 전월 일치).
- 1월은 입춘 전이라 전년도 세운 천간 기준(`sajuYear=year-1`). 2~12월은 해당 연도.
- 점수=`favor()`+`mapScore()`(daily 재사용). 월간 십신→테마(`SIPSIN_THEME`). best(≥58)·caution(≤44) 달 추출. 세운(연간)도 동일 방식.
- `YearlyFortune.tsx`: 올해/내년 토글, 프리미엄 아니면 teaser+`onLocked()`(Paywall). page에서 DailyFortune 바로 아래 배치. `reqBody` prop으로 AI 총평 호출.
- **AI 총평**: `/api/yearly-reading`(스트리밍)이 saju+yearly 서버 재계산 후 `buildYearlySystem/User`(llmPrompt.ts)로 선배톤 5~7문장. 규칙 점수·좋은달/조심달을 사실로 고정(달 숫자만 인용). 프런트는 연도별 캐시(`comments[year]`), body에 `targetYear` 전달. 마커 없는 plain 스트림.
- `scripts/_yearly_test.ts`도 임시잔재(내용 비움, 삭제무방). `_daily_test.ts`와 동일.

## 작명/개명 설계 메모
- `computeNaming(result)`: 발음오행(초성→오음→오행) 기반. 한글 완성형 초성 분해(`(code-0xAC00)/588`)→`SOUND_OHAENG`. 보완오행=용신프록시(daily와 동일: 신약 인성·신강 식상·중화 재성)+수량 최소 오행. 추천 초성/음절 + 입력이름 소리오행 분석/판정. 무료(클라 전용 `NamingCard.tsx`).
- 발음오행 학파: 아ㄱㅋ=목, 설ㄴㄷㄹㅌ=화, 순ㅁㅂㅍ=수, 치ㅅㅈㅊ=금, 후ㅇㅎ=토(훈민정음 후음=토 학파). 한자 자원오행·획수는 미포함(전문 작명 권유 문구로 면책).

## 음양력 변환 설계 메모 (lunar.ts)
- **테이블 없이 천문계산으로 직접 구성**: Meeus ch.49로 합삭(삭) JDE(TT) 계산→KST 변환→삭이 든 KST 달력일이 음력 월의 1일. 동지(270°)를 품은 달=음력 11월로 고정, **무중치윤**(중기 없는 첫 달=윤달)으로 13개월 해 처리.
- **핵심 버그수정**: 무중월 판정을 삭'순간' 황경이 아니라 **중기의 '날짜'가 그 달에 드는지**로 해야 정확(자정 경계 케이스). `zhongqiDayJDN()`로 13개 중기 날짜 구해 day-membership 판정.
- **검증 완료**: 왕복(solar→lunar→solar) 2000~2030 **1116/1116**, 윤달 2020윤4·2023윤2·2025윤6 정확, 설날·대보름 일치. (2027 설날은 삭이 KST 02-07 00:56라 02-07이 정답 — 엔진이 맞음)
- **UI**: 메인 폼 양력/음력 토글 + 윤달 체크. 음력 제출 시 `lunarToSolar`로 변환→폼을 양력으로 갱신+토스트로 "음력→양력" 표시→이후 모든 호출은 양력. `reqBody(override?)`로 변환 양력 주입. 시각(시·분)은 음양력 무관 동일.
- `solarToLunar`도 export(표시/검증용, 현재 UI 미사용).

## 인생 가이드북 PDF 설계 메모
- 추가 의존성 없이 **인쇄 최적화 뷰 + `window.print()`**(사용자가 'PDF로 저장' 선택)로 구현. `GuidebookPrint.tsx`(표지·명식·핵심지표·풀이·운흐름·맺음말), `result`+`ai`(있으면 AI풀이 우선) 기반.
- 화면엔 `display:none`, `@media print`에서 `.wrap > *:not(.guidebook-print){display:none}`로 앱 UI 숨기고 가이드북만 A4 출력(라이트 테마·골드 포인트). GuidebookPrint는 `main.wrap`의 직속 자식이어야 함(현재 result fragment 안, fragment는 DOM 무영향이라 직속).
- 버튼: 프리미엄 카드 내 "📕 인생 가이드북 PDF로 저장" → premium이면 `window.print()`, 아니면 Paywall. 샌드박스에선 인쇄 시각검증 불가(빌드·번들 포함까지 확인).

## 명식 고도화 설계 메모
- `computeHapchung(pillars)`(hapchung.ts): 4기둥 간지 인덱스만으로 천간합(5)·지지 육합(6)·삼합/반합(4국, 2개↑=반합)·충(인덱스차6)·형(삼형 寅巳申·丑戌未, 자형, 子卯상형)·해(6쌍) 스캔. 위치(년월일시) 라벨+tone(good/bad/neutral). 검증: 1995-08-15例 인해육합·인신충·신자진반합·인사신형·신해해 등 정통과 일치.
- 명식 카드에 음력 생일(`solarToLunar`) 줄 + 합충 섹션(.hapchung) 렌더. (가이드북 PDF엔 미반영 — 추후 추가 가능)

## PWA 설계 메모
- `app/manifest.ts`(MetadataRoute.Manifest, standalone·테마 #0c0a16) → Next가 `/manifest.webmanifest` 자동 서빙 + head 링크. `layout.tsx`에 `viewport.themeColor`·`appleWebApp`·icons 메타.
- `public/sw.js`: 페이지=네트워크우선+캐시폴백, 정적=캐시우선, `/api/`·외부도메인은 패스(스트리밍 보존). 오프라인 셸 캐시.
- `PwaSetup.tsx`(layout에 마운트): SW등록 + `beforeinstallprompt`로 설치배너(안드/데스크탑), iOS는 '공유→홈화면 추가' 안내. `appinstalled`시 숨김.
- 아이콘: PIL로 생성한 골드 오비트 엠블럼(192/512/maskable-512/apple-180). **설치 테스트는 localhost 또는 HTTPS 필요**(검증: manifest/sw/아이콘 200, head 주입 확인).

## 택일 설계 메모
- `pickAuspicious(result, purpose, year, month)`(auspicious.ts): 그 달 전일 평가. 일진 favor(daily 재사용)+ 일지충(`(일지+6)%12`) 강한 감점(결혼/이사 -55, 여행 -30)+ 일간합(천간합 5쌍) 보너스+ 목적별(이사=손없는날 음력끝자리9/0 +12, 여행=목화 지지, 결정=일간동일). `topAuspicious`는 충경고일 제외 후 점수순 N개→날짜순.
- `AuspiciousDates.tsx`(프리미엄): 목적 칩 5종+이번달/+1/+2 토글. 이번달은 지난 날 제외. 추천일 리스트+피하는날 안내. 검증: 무인일주 일지충(신)인 9·21일 회피, 신약→토화일 고득점, 이사 손없는날 보너스 동작.

## 개운법 설계 메모
- `computeGaeun(result)`(gaeun.ts): 용신프록시(신약 인성·신강 식상·중화 재성) + 부족(최소 count) + 과다(status '과다') 산출. 오행별 색/방위/음식/취미/직업/소품 테이블(RECO). 과다 오행은 줄이라는 주의문.
- `GaeunCard.tsx`(프리미엄): 핵심 기운 배지 + 6항목 처방 표 + 보조오행 + 과다주의. NamingCard 아래 배치.
